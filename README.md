# Creative Room Analytics

Інтерактивний дашборд для Notion бази `Design Room` з автоматичною синхронізацією даних через GitHub Actions.

## Сторінки

- **`index.html`** — основний дашборд (Chart.js, світла тема): Огляд / Дизайнери / Всі крео. Дані підтягуються автоматично з `assets/latest-data.json`.
- **`old.html`** — попередня версія дашборда (темна тема, `app.js` + `styles.css`), збережена як резерв. Підтримує ручне завантаження CSV.

## Що вміє основний дашборд

- **Автоматичне підтягування даних**: при відкритті завантажує `assets/latest-data.json`, який щодня оновлюється з Notion через GitHub Actions. У шапці видно дату останнього оновлення.
- **Огляд**: KPI (таски, крео, виконано, активні, темп тижня), крео по тижнях, тип креатива (з нуля vs адаптація), лідерборд дизайнерів, швидкість виконання, топ баєрів і GEO, пріоритети, популярні категорії.
- **Дизайнери**: вибір дизайнера й місяця, перемикач гранулярності **День / Тиждень / Місяць**, тренд ▲/▼ до попереднього періоду. **Клік по стовпчику на графіку** фільтрує таблицю тасок до цього дня/тижня/місяця, кнопка `✕ Reset` повертає повний список.
- **Всі крео (таблиця)**: пошук + фільтри по дизайнеру, баєру, GEO, статусу, типу і пріоритету.
- Обсяги (крео) рахуються тільки по **виконаних** тасках; `eff_date` = дата видачі → взяття в роботу → створення.

## Автоматична синхронізація з Notion

Workflow [.github/workflows/sync.yml](.github/workflows/sync.yml) щодня о 06:00 UTC (та вручну через `Run workflow` в Actions):

1. Запускає `scripts/sync_notion.py`, який тягне всі сторінки бази через Notion API.
2. Записує результат у `assets/latest-data.json`.
3. Комітить файл, якщо дані змінилися.

Потрібні секрети репозиторію (`Settings → Secrets and variables → Actions`):

- `NOTION_TOKEN` — інтеграційний токен Notion (інтеграцію треба додати до бази через `Connections`).
- `NOTION_DATABASE_ID` — ID або повне посилання на базу.

`Дата створення` береться з системного поля Notion `created_time`, якщо однойменної властивості в базі немає.

## Пріоритет джерел даних

Основний дашборд (`index.html`) читає лише `assets/latest-data.json`. Стара версія (`old.html`) додатково підтримує: кеш браузера від ручного CSV та демо-CSV.

## Як запускати локально

```bash
cd crm_project
python -m http.server 8080
```

Потім відкрити `http://localhost:8080`. Пряме відкриття `index.html` може працювати не у всіх браузерах, тому локальний сервер надійніший.

## GitHub Pages

Проект — статичний сайт:

1. `Settings → Pages` → `Deploy from a branch`.
2. Гілка `main`, папка `/ (root)`.
3. Після кожного авто-коміту синхронізації Pages передеплоїться сам, і всі відвідувачі бачать свіжі дані.

## Ручне завантаження CSV (через old.html)

1. Відкрийте `old.html`, експортуйте базу з Notion у CSV.
2. Натисніть `Завантажити CSV` (можна кілька файлів одразу).
3. `Зберегти snapshot` завантажить `latest-data.json`, який можна закомітити в `assets/` вручну.

## Дані та приватність

CSV-файли з Notion не комітяться в репозиторій (ігноруються через `.gitignore`).

`assets/latest-data.json` комітиться автоматично — усі, хто має посилання на сайт, бачать ці дані. Якщо в базі є чутлива інформація, не публікуйте репозиторій/сайт публічно.

При ручному завантаженні CSV файл обробляється локально в браузері й нікуди не відправляється.
