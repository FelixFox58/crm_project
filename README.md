# MounthAnalytic

Інтерактивний дашборд для Notion бази `Design Room` з автоматичною синхронізацією даних через GitHub Actions.

## Що вже є

- **Автоматичне підтягування даних**: дашборд при відкритті сам завантажує `assets/latest-data.json`, який щодня оновлюється з Notion через GitHub Actions.
- Фільтр по датах: усе, конкретний місяць, довільний діапазон днів.
- Вибір поля дати: `Дата створення`, `Дата взяття в роботу`, `Дата видачі` (якщо у вибраному полі немає дат, дашборд сам переключиться на найбільш заповнене).
- Аналітика по дизайнерах, баєрах, статусах, GEO, типах креативів і пріоритетах.
- KPI: кількість задач, сума `К-сть`, частка виконаних, середній цикл від взяття в роботу до видачі.
- Пошук по тексту задач і таблиця відфільтрованих задач.
- Експорт поточного фільтру назад у CSV.
- Деталізація по дизайнерах: клік по дизайнеру відкриває задачі, теми, формати та кількість креативів.
- **Активність по днях у деталізації дизайнера**: клік по стовпчику дня показує таски саме цього дня, кнопка `Reset` повертає повний список.
- Ручне завантаження CSV-експорту Notion як резервний варіант (кешується в браузері).

## Автоматична синхронізація з Notion

Workflow [.github/workflows/sync.yml](.github/workflows/sync.yml) щодня о 06:00 UTC (та вручну через `Run workflow` в Actions):

1. Запускає `scripts/sync_notion.py`, який тягне всі сторінки бази через Notion API.
2. Записує результат у `assets/latest-data.json`.
3. Комітить файл, якщо дані змінилися.

Потрібні секрети репозиторію (`Settings → Secrets and variables → Actions`):

- `NOTION_TOKEN` — інтеграційний токен Notion (інтеграцію треба додати до бази через `Connections`).
- `NOTION_DATABASE_ID` — ID або повне посилання на базу.

`Дата створення` береться з системного поля Notion `created_time`, якщо однойменної властивості в базі немає.

## Пріоритет джерел даних при відкритті

1. `assets/latest-data.json` (авто-синхронізація) — основне джерело.
2. Кеш браузера від ручного завантаження CSV (офлайн-резерв).
3. Демо-CSV, якщо є.

## Як запускати локально

```bash
cd crm_project
python -m http.server 8080
```

Потім відкрити `http://localhost:8080`. Пряме відкриття `index.html` може працювати не у всіх браузерах, тому локальний сервер надійніший.

## GitHub Pages

Проект — статичний сайт:

1. `Settings → Pages` → `Deploy from a branch`.
2. Гілка `main`, папка `/ (root)`.
3. Після кожного авто-коміту синхронізації Pages передеплоїться сам, і всі відвідувачі бачать свіжі дані.

## Ручне завантаження CSV (резервний варіант)

1. Експортуйте базу з Notion у CSV.
2. Натисніть `Завантажити CSV` (можна кілька файлів одразу).
3. Дані закешуються в браузері; кнопка `Скинути кеш` повертає до автоматичного снапшота.
4. `Зберегти snapshot` завантажить `latest-data.json`, який можна закомітити в `assets/` вручну.

## Дані та приватність

CSV-файли з Notion не комітяться в репозиторій (ігноруються через `.gitignore`).

`assets/latest-data.json` комітиться автоматично — усі, хто має посилання на сайт, бачать ці дані. Якщо в базі є чутлива інформація, не публікуйте репозиторій/сайт публічно.

При ручному завантаженні CSV файл обробляється локально в браузері й нікуди не відправляється.
